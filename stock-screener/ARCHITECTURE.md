## 2. State Management & Store Architecture

Global state is divided into focused slices managed by **Zustand**. Slices are organized to prevent unneeded re-render propagation.

```text
                 +-------------------+
                 |  useStockStore    | <---+ (Fast O(1) Map cache for 5000+ stocks)
                 +---------+---------+
                           |
       +-------------------+-------------------+-------------------+
       |                   |                   |                   |
+------+------+     +------+------+     +------+------+     +------+------+
| FilterStore |     |ScreenerStore|     | ChartStore  |     |WatchlistStor|
| (AST rules, |     | (Sort order,|     | (Selected   |     | (Lists cache|
| search text)|     | column hide)|     | indicator)  |     | localstorage|
+-------------+     +-------------+     +-------------+     +-------------+
       |                   |                   |                   |
       +-------------------+-------------------+-------------------+
                           |
                 +---------v---------+
                 |   useAlertStore   | (Dynamic alert monitoring logs queue)
                 +---------+---------+
                           |
                 +---------v---------+
                 | useWebSocketStore | (Connection state, throughput metrics)
                 +-------------------+
```

### Stores Index & Selectors
- **`useStockStore`**: Holds a Javascript `Map<string, Stock>` representing the stock database. Features O(1) lookups and a fast batch writer `updateStocksBatch`.
- **`useFilterStore`**: Manages filter inputs, an array of AST rules, and preset configurations.
- **`useScreenerStore`**: Manages column sorting, column visibility states, and multi-row selection records.
- **`useChartStore`**: Holds chart attributes: active timeframe, visible overlays, and chart theme.
- **`useWatchlistStore`**: Handles user watchlist mapping (`Record<string, string[]>`) persisted to `localStorage`.
- **`useAlertStore`**: Stores historical alerts generated by price/volume threshold exceptions.
- **`useWebSocketStore`**: Manages the connection status and monitoring diagnostics (e.g. latency, throughput).

## 3. Real-Time Data Flow (WebSocket Engine)

The mock WebSocket pipeline uses a double-buffered architecture to allow 5,000+ records to update in real time at 60 FPS without layout thrashing.

```text
    [websocketService (GBM Simulation)]
                  |
                  | Ticks (Batches of 15 stocks every 250ms)
                  v
       [useWebSocket Hook Buffer]
                  |
                  | Accumulates in patchQueue.current
                  | Triggered by requestAnimationFrame Loop
                  | Throttled at 100ms
                  v
       [useStockStore.updateStocksBatch]
                  |
                  | Atomic state commit (Single React update)
                  v
            [StockTable]
       (Re-evaluates viewport)
                  |
                  v
         [FlashCell component]
   - Compares previous price vs current price
   - Mutates classList directly via element refs
   - Skips React re-rendering for cells
```

## 4. JIT AST Filter Engine Architecture

Filters are organized into a Abstract Syntax Tree (AST) which is JIT-compiled into a short-circuiting matcher.

```text
         +-------------------------------------------------+
         | LogicalGroup (AND/OR)                           |
         |  - Rule 1: Numeric (Price > 100)                |
         |  - Rule 2: Categorical (Sector IN [Tech, Fin])  |
         |  - Rule 3: Text (Name Contains "Apple")         |
         +------------------------+------------------------+
                                  |
                                  v
                       [compileAST Compiler]
                                  |
            Does serialization match key in cache?
                         /               \
                       YES                NO
                       /                    \
            [Return cached function]   [Build short-circuit function]
                                        - Check Rule 1 (false? Exit AND early)
                                        - Check Rule 2 (false? Exit AND early)
                                        - Check Rule 3 (false? Exit AND early)
                                        - Save to cache Map
                                  |
                                  v
                     [Compiled Boolean Evaluator]
                                  |
                                  v
                      [Array.filter evaluation]
                       (Sub-200ms over 5000)
```

## 5. Synced Chart Architecture

The chart panel leverages two stacked lightweight-charts canvases.

```text
+-------------------------------------------------------+
|  Primary Chart Canvas (ChartPanel)                    |
|  - CandlestickSeries (OHLC bars)                      |
|  - LineSeries (SMA, EMA, VWAP, Bollinger Bands)      |
|  - HistogramSeries (Volume bars scaled to lower 20%)  |
+--------------------------+----------------------------+
                           |
         ZOOM / PAN        | Visible Time Range Change
         CROSSHAIR SYNC    | Event Listener
                           v
+--------------------------+----------------------------+
|  Secondary Chart Canvas (RSI Oscillator)              |
|  - LineSeries (Calculated RSI line)                   |
|  - Overbought Limit line (70 threshold)               |
|  - Oversold Limit line (30 threshold)                 |
+-------------------------------------------------------+
```

### Manual Technical Indicators Engine (`src/utils/indicators.ts`)
When historical price candles load via React Query, a mathematical engine computes TA overlays:
- **Simple Moving Average (SMA)**: Sliding sum divider.
- **Exponential Moving Average (EMA)**: Wilder multiplier applied to previous points.
- **Relative Strength Index (RSI)**: Smooth average gain/loss calculation.
- **Bollinger Bands**: SMA basis +/- Standard Deviation window.
- **VWAP**: Cumulative Typical Price volume divider.
- **Volume Profile**: Bin frequency accumulator based on typical session values.
